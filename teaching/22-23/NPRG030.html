  • ošetření chyb, tzv. výjimky: Za zachycení stojí jen ty chyby, u kterých víme, jak chceme, aby program pokračoval. Mnohdy je pro nás výhodnější při chybě program prostě ukončit místo zachycování výjimek. [seznam built-in výjimek]
    • Ukázka použití try-except k ošetření chyby. Jaká výjimka by se měla zachytit u proměnné c?
    try:
      a = int('pokus')
    except ValueError:
      print('Toto není číslo.')

    try:
      b = 7 / 0
    except ZeroDivisionError:
      print('Dělíte nulou.')

    try:
      c = len(9)
    except:
      print('??')

    Rozšířené použití try-except k ošetření chyby a dalšímu běhu programu.
    a = 7
    b = 0
    try:
      c = a / b
      print(c)
    except ZeroDivisionError:
      print('Dělí se nulou!')
    else:
      print('Tohle se provede, pokud chyba nenastane.')
    finally:
      print('Tohle se provede vždycky.')
    Vlastní vyvolání výjimky pomocí raise místo return.
    def obsah_ctverce(strana):
      if strana > 0:
        return strana ** 2
      else:
        raise ValueError(f'Strana musí být kladná, číslo {strana} kladné není!')

  • testování
    • způsob, jak automaticky ověřit, zda vytvořený program funguje dle požadavků; vytváří se program, který testuje funkci jiného programu
    • automatické testy se mohou zkpouštět opakovaně, např. při úpravě programu dává smysl ověřit si, že úprava programu nezpůsobila problémy ve funkčnosti
    • v Pythonu existuje knihovna unittest, díky níž lze vytvářet testy, ale na ovládání je mnohem snazší knihovna pytest, která se však musí doinstalovat
      • python3 -m pip install pytest
    • tvorba testů má své konvence: vytvářené testy by měly mít předponu test_, k vyhodnocení se používá klíčové slovo assert, testy se většinou udržují ve speciálním samostatném souboru
      • Ukázka jednoduchého testu uloženého v souboru test_secteni.py.
      def secti(a, b):
        return a + b

      def test_secti():
        assert secti(1, 2) == 3

      Příkaz assert ve výjimkách funguje v podstatě jako:

      if not (a == b):
        raise AssertionError('Test selhal!')

      Test se spouští z příkazového řádku pomocí:

      python3 -m pytest -v test_secteni.py

      Vytvořený test je tzv. pozitivní test, lze ale vytvořit i negativní testy:

      def test_secti2():
        assert secti(1, 2) != 4
      Rozdělte soubor test_secteni.py na dva soubory tak, aby byly testy a kód programu zvlášť, a přitom šlo kód programu testovat.
      Ukázka parametrického testu.
      import pytest

      @pytest.mark.parametrize(
        ['a', 'b'], [(10, 1), (11, 2), (12, 3), (13, 4), (14, 5)]
      )
      def test_secti(a, b):
        assert secti(a, b) == a + b

  • čtení a zápis z/do textového souboru
    • Jak číst a zapisovat do souboru? Co znamená klíčové slovo with?
    # starší způsob čtení
    file = open('soubor.txt', 'r')
    obsah = file.read()
    file.close()

    # modernější způsob čtení
    with open('soubor.txt', mode='r', encoding='U8') as file:
      obsah = file.read()

    Čtení ze souboru lze také následovně:

    • file.read() (příp. file.read(5)),
    • file.readline(),
    • prostým cyklem for line in file: print(line)
    # starší způsob zápisu
    file = open('soubor.txt', 'w')
    file.write('text...')
    file.close()

    # modernější způsob zápisu
    with open('soubor.txt', mode='w', encoding='U8') as file:
      file.write('text...') # nebo: print('text...', file=file)

    * Parametr mode může nabývat několika hodnot, např. w (write), r (read), b (binary mode), a (append). Stejně tak encoding.

    with : zajišťuje, že nezůstane soubor otevřený a že čtení i zápis proběhnou hladce [popis z dokumentace]
